在实际设备（HTC HERO@sprint）上运行Android程序 | Android Develop - Using Hardware Devices

今天开始读Android基础教程（第3版），跑图书馆呆了一个下午。

晚上回实验室开始动手。

这里有详细的步骤：http://developer.android.com/guide/developing/device.html

首先需要在AndroidManifest.xml中的<application>元素下增加这一行：android:debuggable=”true”

然后，在手机 设置 应用程序 中允许未知源软件的安装，并打开USB调试开关

最后，也是最终重要的，需要下载一个usb driver来让adb能识别你的设备，而这个驱动则是厂商相关的，需要根据不同手机下载不同的驱动。

我的是HTC HERO@sprint，下载链接为http://www.htc.com/us/support/hero-sprint/downloads/

不过并没有找到什么usb driver的驱动，后来了解到HTC SYNC中包含了这个驱动，所以下载HTC SYNC并安装，也就安装好了usb driver。

这个时候，连接手机，用adb devices命令就可以检查是否成功识别的设备了。

然后在Eclipse中运行Android应用程序，在设备选择的时候可以看到连接的手机，选择它，程序便会在手机上加载并运行了。
